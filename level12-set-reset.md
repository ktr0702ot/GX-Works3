# Level12 SET・RSTによる状態保持

## 概要

PLCで状態を保持する方法として、

- SET命令
- RST命令
- SET/RSTによる運転状態の保持
- 異常発生時のリセット
- SETとRSTが同時に成立した場合の動作

について学習した。

以前作成した自己保持回路との違いも確認し、設備の運転・停止を想定したラダーを作成した。

---

## SET命令

SET命令は、条件が成立したときに指定したデバイスをONにし、その後条件が不成立になってもON状態を保持する。

今回、以下の回路を作成した。

```text
X40
↓
SET M50
```

動作：

```text
X40 ON
↓
SET M50実行
↓
M50 ON
```

その後X40をOFFにしても、

```text
X40 OFF
↓
M50 ONを保持
```

となることを確認した。

---

## RST命令

SETによってON状態を保持しているデバイスをOFFにするため、RST命令を使用した。

以下の回路を作成した。

```text
X41
↓
RST M50
```

動作：

```text
X41 ON
↓
RST M50実行
↓
M50 OFF
```

X41をOFFに戻しても、

```text
X41 OFF
↓
M50 OFFを維持
```

となる。

---

## SET・RSTの基本動作

今回作成した回路は以下。

```text
X40 ───────── SET M50

X41 ───────── RST M50
```

動作：

```text
X40 ON
↓
SET M50
↓
M50 ONを保持

X41 ON
↓
RST M50
↓
M50 OFF
```

---

## 出力との連動

M50を設備の運転状態として考え、Y50を設備運転出力として使用した。

以下の回路を追加した。

```text
M50 ───────── ( Y50 )
```

全体の動作：

```text
X40 ON
↓
SET M50
↓
M50 ON
↓
Y50 ON
```

X40をOFFにしても、

```text
M50 ONを保持
↓
Y50 ONを維持
```

する。

停止操作を行うと、

```text
X41 ON
↓
RST M50
↓
M50 OFF
↓
Y50 OFF
```

となる。

---

## 自己保持回路との違い

以前、接点とコイルを使用した自己保持回路を作成した。

自己保持回路では、自分自身の接点を使用してON状態を維持する。

イメージ：

```text
       運転開始
     ┌──[ ]──┐
─────┤       ├──── 異常なし ────( 運転状態 )
     └─[運転]┘
```

運転中に「異常なし」の条件が不成立になると、

```text
異常発生
↓
異常なし条件が不成立
↓
自己保持回路が切れる
↓
運転状態OFF
```

となる。

その後異常が解除されても、

```text
異常解除
↓
自己保持はすでに解除済み
↓
運転状態OFFのまま
↓
再度運転開始操作が必要
```

となる。

---

## SET使用時の注意

SET命令の場合、一度SETされたデバイスはRSTされるまでON状態を保持する。

そのため、

```text
SET M50
↓
M50 ON
```

となった後に異常が発生しても、

```text
RST M50
```

を実行するロジックがなければM50はON状態を保持する。

設備制御では、異常発生時に安全に停止させるためのRST条件を設計する必要がある。

---

## 異常発生によるRST

異常発生時にもM50をOFFにするため、

```text
X42：異常発生
```

を追加した。

停止条件は、

```text
X41：停止操作
OR
X42：異常発生
```

とした。

ラダーイメージ：

```text
       X41
    ┌──[ ]──┐
────┤       ├──── RST M50
    └──[X42]┘
```

X41とX42を並列に配置することで、

```text
X41 ON
OR
X42 ON
↓
RST M50
```

となる。

---

## 異常発生時の動作確認

まず正常状態で運転した。

```text
X40 ON
↓
SET M50
↓
M50 ON
↓
Y50 ON
```

その後、

```text
X42 ON
```

として異常発生状態を再現した。

結果：

```text
X42 ON
↓
RST M50
↓
M50 OFF
↓
Y50 OFF
```

となることを確認した。

---

## SETとRSTが同時に成立した場合

次に、

```text
X40 ON
X42 ON
```

として、

```text
SET M50
RST M50
```

が同時に実行される状態を作成した。

今回のラダーでは、

```text
SET M50
↓
RST M50
```

の順番で実行される。

そのため、

```text
X40 ON
↓
SET M50
↓
M50 ON

同一スキャン内で

X42 ON
↓
RST M50
↓
M50 OFF
```

となり、最終的にM50はOFFとなった。

今回の回路では、

```text
運転開始
よりも
停止・異常
を優先
```

する動作となることを確認した。

---

## 実務での考え方

設備制御では、

```text
運転要求
↓
SET
↓
運転状態保持
```

だけではなく、

```text
停止操作
異常発生
インターロック不成立
非常停止
```

などの条件で安全に運転状態を解除する必要がある。

SET/RSTを使用する場合は、

> どの条件でSETされるか

だけでなく、

> どの条件でRSTされるか

を確認することが重要である。

---

## 学んだこと

Level12では以下を学習した。

- SET命令によるON状態の保持
- RST命令による状態解除
- M50とY50の連動
- 自己保持回路とSET/RSTの違い
- 異常発生時のRST処理
- OR条件による複数のRST条件
- SETとRSTが同時成立した場合の動作
- 停止・異常を優先する考え方

設備制御では、

```text
運転開始
↓
SET
↓
運転状態保持

停止・異常
↓
RST
↓
運転状態解除
```

というロジックを理解することが重要である。
